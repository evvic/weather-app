Weather App

How Weather Signals Are Recorded and Displayed

Using weather measuring devices available in the TAMK lab, my group and I wired an Arduino, LCD, ethernet module, and more together to create a functioning device which can take analog signals from the measuring device and ultimately send them to my database to display on this website.

My website then uses created API endpoints from the backend to sort and display the data in meaningful ways.

The frontend of the webpage uses charts and tables to display the data in a sorted way. These charts and tables all are coded with pure JavaScript.

Sending Data

Use Of MQTT Messaging To The Broker

After the Arduino received a weather signal, it translates the signal to an understandable value then formats an MQTT message with the value and datatype (i.e. our group is wind_speed). The message is then sent to the MQTT broker.

The MQTT broker is a VirtualMacine on campus that accepts the MQTT message. Which then runs the C++ gateway code to parse the MQTT message into a JSON formatted message and sends it to my database.

Finally the JSON data is just stored in my database, to be called and sorted on this wesite.
